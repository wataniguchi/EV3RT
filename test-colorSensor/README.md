# claude-colorSensor

libraspike_art_python 経由でカラーセンサーを用いて色判定・ライントレース制御値を取得するスクリプト群。

---

## ファイル構成

| ファイル | 役割 |
|----------|------|
| `collectData.py` | カラーセンサー値（RGB・HSV）を収集してファイルに保存 |
| `identifyColor.py` | HSV値から色を判定し、色変化イベントを検出 |
| `data/color_sensor_data.txt` | 収集済みデータ（書式：CaseID,CaseName,R,G,B,H,S,V） |

## 色判定ロジック（identifyColor.py）

### 判定の優先順位と閾値

```
1. 白   : S < 20  AND  V > 75
2. 青   : H ∈ [205, 220]  AND  S > 50
3. 緑   : H ∈ [140, 155]  AND  S > 60
4. 黄   : H ∈ [35, 65]    AND  V > 88
5. 赤   : (H > 345 or H < 10)  AND  S > 75
6. 黒   : S < 35  （上記すべてに非該当の低彩度）
7. 不明 : それ以外
```

### 閾値設計の根拠

**白の条件を S < 20 に絞った理由**
純白の収集データは S 最大値が 19。外乱光や走行体の揺れで S が 20〜34 に上昇した
読み値（実測で複数確認）は白ではなく黒として扱うべきであることが判明した。

**青の S 閾値を 50 に緩めた理由**
黒-青境界・青-白境界では S が 59〜71 まで低下する（純青は S≈93）。
H 範囲 [205, 220] は他の色と重複しないため、S を下げても誤判定リスクは低い。

**黒を「S < 35 のすべて」とした理由**
外乱光でV（輝度）が 51〜91 まで上昇しても、有色ラインであれば S は必ず高くなる。
有彩色をすべて先にチェックした後、残った低彩度の読み値は黒（またはその境界）と判断して問題ない。

### 実走で確認した「黒に分類すべき境界的な読み値」

| H | S | V | 状況 |
|---|---|---|------|
| 190 | 34 | 57 | 黒-白境界付近 |
| 190 | 20 | 67 | 黒-白境界付近 |
| 186 | 26 | 51 | 黒線上（外乱光でV上昇） |
| 207 | 24 | 91 | 黒線付近（V高いが彩度低い） |

いずれも S < 35 かつ白条件（S < 20 AND V > 75）を満たさないため、正しく黒に分類される。

### ロバスト化：直近5サンプルの多数決

走行体の左右揺れにより、境界区間では黒と有色が交互に出現する。
`classify_robust()` は直近 5 サンプルを集計し、**黒票を除いた多数決**で有色を優先する。
有色が過半数（3票以上）に達しない場合は黒を返す。

## ライントレースのPID制御値

### 推奨：グレースケール輝度 V をそのまま制御値に使う

カラーセンサーをグレースケールモード、またはカラーモードで取得した HSV の V 値を PID 制御の入力とする。

| センサ値 | 色・状況 | V の目安 |
|----------|----------|----------|
| 黒線上 | 黒 | ≈ 26 |
| エッジ（目標位置） | 黒-白境界 | ≈ 62 |
| 白領域 | 白 | ≈ 98 |
| 青線上 | 青 | ≈ 77 |

**目標値（setpoint）の初期値 = 62**（黒と白の中間）

```
偏差 = V - 62
偏差 > 0 → 白側に外れている → 黒線方向に曲がる
偏差 < 0 → 黒線上にいる   → 白側に曲がる
```

### 色相 H ではなく輝度 V を使う理由

- 黒・白・灰色は彩度が低く H が不定のため PID の入力として使えない
- 黒→青→黒の遷移で H が急変し制御が乱れる
- V は黒（≈26）→白（≈98）で単調増加し、青線（≈77）通過時も連続して変化するため制御が乱れない

### 役割分担

| 役割 | 手段 |
|------|------|
| ラインエッジのトレース（PID） | グレースケール V（連続値） |
| 色変化イベントの検出（青・緑・黄・赤の通過判定） | `identifyColor.py` の色判定 |

目標値 62 は実機走行で調整すること。走行速度・センサ取り付け高さ・外光条件によって最適値は変わる。

## 実行方法

```bash
# データ収集
python collectData.py

# 色判定（40ms周期、Ctrl+C で停止）
python identifyColor.py
```
